Files and folders structure

Whole app is divided in to several directories and subdirectories. The whole app consists of Vue components, subcomponents and Django specific python files. The whole directory structure is given below. The work of every component and subcomponent can be easily understood by their name.

P.S. This directory structure is adopted from an open-source template for using Django and Vue.js together.
This is the link for the default template: https://github.com/gtalarico/django-vue-template. 

├── babel.config.js     // Configuration for Babel
├── backend             // Django apps and configuration directory
│  ├── api             // Django api app directory
│  ├── authentication  // Django authentication app directory
│  ├── clients         // Django clients app directory
│  ├── company         // Django company app directory
│  ├── invoices        // Django invoices app directory
│  ├── items           // Django items app directory
│  └──  settings        // Django settings directory both for production and development
├── manage.py           // Django manage.py file
├── package.json        // node package.json file for dependency management
├── Pipfile             // Pipfile for for python package dependency management
├── public              // public directory for vue app
├── requirements.txt    // Python requirements.txt
├── src                 // Vue app source directory
├── templates           // Templates directory for Django app
└── vue.config.js       // Vue configuration file

Setup the app

  • Install Python 3.7.1
  • Install the Python dependencies 
    pip install pipenv
pipenv install
  • Run database migration to create necessary tables and datas.
    python manage.py migrate
  • Set ALLOWED_HOSTS on backend/settings/prod.py.
  • Install npm or yarn
  • Install all the required javascript dependencies 
    yarn install // For yarn
npm install  // For npm
  • Build for Production 
    yarn build  // For yarn
npm run build  // For npm
After building for production it will create a dist directory in the root directory. This directory will be served by the Django app.
P.S. Everytime you change anything in the source code you must build the template again. Otherwise, your changes will not reflect on the template.

Testing Locally

You can test the app locally easily with the below steps:

  • Install Python 3.7, node, yarn/npm.
  • Run the migration using this command: python manage.py migrate
  • Run the local django server: python manage.py runserver
  • Run the local yarn server: yarn serve or npm run serve
P.S. You can test your app using only the django server also. Then you'll have to run yarn build everytime after changing anything inside your app.

Grid system

"Invoisu" is based upon Twitter Bootstrap css framework that is very easy to use and adapt. It follows the mobile first approach to make the website readily available for all devices and screen sizes. Bootstrap grid is based upon 12 columns that are well defined in the bootstrap.css file inside the css folder. You can get to know about the grid system in detail from this url https://getbootstrap.com/docs/4.0/layout/grid/

We used a package called bootstrap-vue to implement the Bootstrap 4 Grid into the template. You can find out more about the package here: https://bootstrap-vue.js.org/

Users

This section contains all the users in the App database.
You can Add, Edit, Delete and also get a PDF view of the User List from this section.

You can access the django views code in the backend/authentication/apiviews.py
You can get a list of all the Users by sending a GET request to this URL: /api/users
You can create a new User by sending a POST request to this URL: /api/users
You can get details of a particular User by sending a GET request to this URL with his/her Username: /api/items/:username
You can update a particular User details by sending PUT request to this URL with his/her Username: /api/items/:username
You can delete a particular User details by sending DELETE request to this URL with his/her Username: /api/items/:username
You can view your Users from your Admin Panel from this URL: #/items.

Users data format(All fields are required): 

{
    "first_name": "",
    "last_name": "",
    "email": "",
    "username": "",
    "password": "",        // Only needed for creating users
    "is_active": false,
    "is_superuser": false
}

Generate Invoice

To generate an Invoice there are 2 pre-requisites:

  1. A client must exist on database.
  2. An item must exist on database.

Then the invoice can be generated easily by pressing the related book icon in the Invoice List Page. It will generate a PDF which can be viewed in the browser before downloading.
